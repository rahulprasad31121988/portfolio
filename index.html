Projects Please find below some examples of world-class products that I have delivered.

BIGIP

BIGIP is an application load balancer that is used to distribute incoming traffic to backend servers using algorithms such as round robin, least connection, IP hash, etc. I was responsible for the configuration interfaces of the product i.e. GUI portal, REST APIs, and command line along with the health monitoring systems for the backend servers and audit and error logging services.

Objectives:

  • Intuitive configuration interfaces with minimal training requirements.
  • Reliable and tamper-proof audit and logging service.
  • Consistent and accurate health monitoring service.

Activities:

  • Requirement gathering through customer interviews and interactions with sales, customer success, engineering, and services teams.
  • Defining product vision and roadmap.
  • Prioritization of features based on business needs and technical dependencies.
  • Work with cross-functional teams such as enterprise architects, designers, documentation and release teams and create alignment among the teams.
  • Work with internal and external stakeholders such as customer success managers, release managers and customers to decide the release schedule.

Evaluation Metrics:

  • Developer time spent configuring the product.
  • Lost audit and error logs.
  • Error rate for health status reporting.

Outcome:

  • 10% reduction in the developer hours spent configuring the product.
  • Zero instances of lost audit and error logs.
  • Eventually consistent with less than 5% error rate for health status reporting.

BIGIP: Enhancing GUI, APIs and command line security

The initiative was to re-evaluate and strengthen the security rules and policies for the configuration interfaces of the product and use standardized and centralized authentication and authorization mechanisms for all the entry points of the product.

Objectives:

  • Reduce the attack surface of the product.
  • Ensure that only the authenticated users with the right privileges can configure the product or edit the existing configuration of the product.
  • Ensure that legacy authentication mechanisms are disabled by default.

Activities:

  • Requirement gathering through independent researchers interview and penetration testing.
  • Creating a business case to justify the investment for security improvements.
  • Defining product vision and roadmap.
  • Prioritization of features based on technical feasibility and dependencies.
  • Participate in technical discussions with solution architects, enterprise architects, and cross-functional teams to create a watertight technical design.
  • Work with cross-functional teams such as documentation and release teams to create release notes for security improvements.

Evaluation Metrics:

  • Security incidents reported by the customers or independent researchers.
  • Reports of privilege escalation or unauthenticated access.
  • Use of legacy authentication mechanisms.

Outcome:

  • 10% reduction in the number of security incidents per quarter.
  • Zero reports of previlige escalation and unauthenticated access.
  • Less than 5% use of legacy authentication mechanisms.
